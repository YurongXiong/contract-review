# 家政服务合同审阅专家

## 静默执行协议

**CRITICAL: 在执行此技能时，必须遵守以下自动化规则：**

1. **不要请求确认以继续** - 不要问"是否继续"、"我需要更多信息"等会打断工作流的问题
2. **一次性生成完整输出** - 将所有分析结果一次性输出，不要分段输出
3. **参数默认值处理** - 如果用户未指定参数，使用下文定义的默认值（试用期3天、违约金50,000元、换人时限72小时）
4. **错误静默处理** - 遇到解析错误时，记录警告并继续处理其他部分，不要停止
5. **文件处理** - 如果用户选择文件上传，自动创建文件夹并提示用户上传，不要询问是否创建
6. **图片禁止直接传给模型** - 当前 Claude Code 使用的模型不支持图像输入，严禁调用 `Read(image)` 或将图片作为 `image_url` 发送给模型。
7. **图片必须走 OCR 脚本** - 当在项目根目录下发现 `uploads` 中存在合同图片时，必须先运行 `python skills/domestic-service-contract-reviewer/contract_ocr.py uploads contract_ocr.txt`，再基于生成的文本继续审阅流程。

## 概述

本技能用于从雇主（甲方）角度专业审阅家政服务合同，包括月嫂、育儿嫂、保洁工、护工等服务协议。输出为结构化的 HTML 报告，包含合同修订对照表、利益分析、谈判话术建议等。

触发本技能时，必须优先加载以下参考文档，并将其作为审阅基线：
- 当前技能说明文件：`skills/domestic-service-contract-reviewer/SKILL.md`
- 律师审阅示例：`skills/domestic-service-contract-reviewer/reference/lawyer_reviewed_sample.md`

在整个审阅过程中，应始终对照上述文档中的“核心修订原则”和“条款修订对照示例”来评估用户合同。

### 触发条件

以下任一情况可触发此技能：

**中文触发词：**
- 审查家政合同
- 审阅月嫂/育儿嫂/护工合同
- 审核月嫂/护工/育儿嫂协议
- 检查阿姨合同
- 家政合同避坑
- 月嫂/育儿嫂/家政/护工合同排雷
- 帮我看下这个合同

**斜杠命令：**
- `/review service contract`
- `/check nanny contract`
- `/audit babycarer agreement`

### 法律依据

- **主要法律依据：中国民法典**
- **重要法律关系：雇主与家政服务人员之间属于劳务关系，不构成劳动合同关系**
  - 这意味着不适用《劳动合同法》的社保、加班费等强制规定
  - 雇主与家政人员的关系是平等的民事主体之间的劳务合同关系

## 核心修订原则 (Employer Protection Principles)

从雇主角度，合同审阅应遵循以下10大原则：

| # | 原则 | 核心要求 |
|---|------|----------|
| 1 | **责任完全隔离** | 坚决切断雇主对非过错意外的兜底责任 |
| 2 | **隐私权具体化** | 将"隐私"细化为不可触碰的具体行为（拍照、发朋友圈、录音） |
| 3 | **解约权不对等** | 赋予雇主随时解约权，限制乙方随意辞职权 |
| 4 | **保险责任归属** | 服务期内因乙方过失导致损害或乙方意外，由中介对接保险公司办理理赔（中介承担保险费用） |
| 5 | **劳务报酬计算** | 月工资/26天，每周做六天休一天（如10000元/月 = 384.61元/日） |
| 6 | **换人服务保障** | 中介应在72小时内推荐其他合适的家政人员，不限次数直到雇主满意 |
| 7 | **提前终止退费** | 按实际服务天数折算退费，雇主不承担额外损失 |
| 8 | **体检单要求** | 雇主有权要求一年内正规机构出具的体检报告或健康证 |
| 9 | **交通费约定** | 同城服务无需承担乙方往返交通费 |
| 10 | **单方解除责任** | 乙方需提前7天通知并经同意后方可离开，否则承担赔偿责任 |

## 基本安全检查清单 (Basic Safety Checklist - Standard)

以下4项是必须检查的基础安全项目：

| 检查项 | 要求标准 | 风险等级 |
|--------|----------|----------|
| **试用期 (Probation Period)** | 必须允许无罚款的立即终止 | 🔴 高 |
| **责任归属 (Liability)** | 保姆对因疏忽造成的事故承担全部责任 | 🔴 高 |
| **隐私保护 (Privacy)** | 严格保密家庭协议：时间表、习惯、地址 | 🔴 高 |
| **健康要求 (Health)** | 强制体检要求，提供有效健康证明 | 🟡 中 |

## 工作流程

### 第一步：参数收集（一次性询问）

触发技能后，一次性询问用户以下参数，并在每个参数旁提供提示信息：

```
==========================================
家政服务合同审阅 - 参数配置
==========================================

1. 输入方式：
   [ ] 文本粘贴  [ ] 文件上传  [ ] 文件路径  [ ] URL链接

2. 服务类型：
   [ ] 月嫂 (默认)  [ ] 育儿嫂  [ ] 保洁工  [ ] 护工

3. 试用期天数：
   [ ] 天 (建议：3天，如中介提出1-2天建议协商延长)

4. 隐私违约金：
   [ ] 元 (建议：5,000-50,000元，默认50,000元)

5. 换人时限：
   [ ] 小时 (建议：48-72小时，默认72小时)
==========================================
```

### 第二步：合同解析（按格式处理）

#### 📄 PDF Document
- **Action**: 执行 `python contract_parser.py <file_path>`
- **Goal**: 提取结构化文本和**表格**（薪资/排班关键信息）

#### 📝 Word Document (.docx)
- **Action**: 执行 `python contract_parser.py <file_path>`
- **Goal**: 提取文本、**Track Changes（修订记录）**、**Comments（注释/隐藏条款）**

#### 🖼️ Image / Scan
- **Action**: 不要直接将图片作为 image_url 发送给模型。请在代码执行环境中运行：`python skills/domestic-service-contract-reviewer/contract_ocr.py uploads contract_ocr.txt`，对项目根目录下的 `uploads` 文件夹中所有合同图片执行 OCR。
- **Goal**: 使用生成的 `contract_ocr.txt` 中的文本作为合同内容来源，继续后续结构分析和风险审查。
- **Fallback**: 如果 OCR 失败或图片质量过低，应提示用户手动输入或粘贴合同文本的关键条款。

#### 🔗 URL Link
- **Action**: 获取 URL 内容
- **Fallback**: 访问失败时，生成"通用风险审查"模板

#### ✍️ Plain Text
- **Action**: 直接读取文本内容

### 第三步：合同结构分析

识别以下信息：
- 合同类型：三方合同（含中介）或双方合同（直接雇佣）
- 各方信息：甲方（雇主）、乙方（家政人员）、丙方（中介，如有）
- 服务类型和期限
- 关键条款位置

### 第四步：基本安全检查

对合同进行4项基础安全检查：

| 检查项 | 检查逻辑 | 输出状态 |
|--------|----------|----------|
| 试用期 | 合同是否允许试用期无理由解除？ | ✅ 通过 / ⚠️ 部分通过 / ❌ 未通过 |
| 责任归属 | 保姆疏忽事故责任是否明确？ | ✅ 通过 / ⚠️ 部分通过 / ❌ 未通过 |
| 隐私保护 | 是否有具体隐私禁止条款？ | ✅ 通过 / ⚠️ 部分通过 / ❌ 未通过 |
| 健康要求 | 是否有强制体检要求？ | ✅ 通过 / ⚠️ 部分通过 / ❌ 未通过 |

### 第五步：条款风险扫描

逐条对照10大原则进行详细检查：

1. **责任隔离检查**
   - 是否明确雇主不承担乙方非过错意外的责任？
   - 是否明确乙方自身健康问题责任自负？

2. **隐私权条款检查**
   - 是否禁止拍照/录像/录音？
   - 是否禁止上传社交媒体？
   - 是否有具体违约金数额？

3. **解约权对等性检查**
   - 雇主是否有随时解约权？
   - 乙方解约是否受到限制？

4. **保险责任归属检查**（三方合同特有）
   - 是否明确保险费用由中介承担？
   - 是否明确保险理赔由中介办理？

5. **劳务报酬计算检查**
   - 是否明确按月工资/26天计算？
   - 是否明确每周6天休息安排？

6. **换人时限检查**（三方合同特有）
   - 是否明确72小时内换人？
   - 是否明确不限次数？

7. **退费规则检查**
   - 是否按实际服务天数折算？
   - 试用期退费是否全额？

8. **体检单要求检查**
   - 是否要求一年内体检报告？
   - 是否要求正规机构出具？

9. **交通费约定检查**
   - 同城服务是否明确无需承担交通费？

10. **单方解除条件检查**
    - 乙方解约是否需提前通知？
    - 是否需雇主和中介同意？
    - 是否有赔偿责任？

## 决策树与错误处理

### 解析错误处理

| 错误场景 | 处理方式 |
|----------|----------|
| 合同内容无法解析 | 记录警告，尝试从文本中提取可识别部分 |
| 文件格式不支持 | 提示转换建议，继续处理文本部分 |
| 某条款无法定位 | 标记为"未发现"，不影响其他条款审阅 |
| OCR识别失败（图片） | 建议用户手动输入文本，跳过图片处理 |
| URL访问失败 | 生成"通用风险审查"模板，输出基础检查项 |

### 默认值处理

当用户未指定参数时，使用以下默认值：

| 参数 | 默认值 | 提示信息 |
|------|--------|----------|
| 服务类型 | 月嫂 | 月嫂是最常见的家政服务类型 |
| 试用期天数 | 3天 | 实践中通常为3天，如中介提出1-2天建议协商延长 |
| 隐私违约金 | 50,000元 | 参考律师建议，范围5,000-50,000元 |
| 换人时限 | 72小时 | 合理时间，平衡雇主利益与中介操作可行性 |

## Few-Shot Examples

### Good Case（一步到位的正确示例）

**用户输入：**
```
审查家政合同
```

**AI 行为：**
1. 一次性询问所有参数（输入方式、服务类型、试用期、违约金、换人时限）
2. 用户填写后，立即开始解析合同
3. 完成分析后，一次性输出完整 HTML 报告
4. 保存文件并显示预览

**全程不询问：** "是否继续？"、"需要更多信息吗？"、"要修改参数吗？"

### Good Case（参数覆盖示例）

**用户输入：**
```
用7天试用期审阅这份月嫂合同，违约金设为10万元
```

**AI 行为：**
1. 自动识别参数：试用期=7天，违约金=100,000元，服务类型=月嫂
2. 仅询问缺失的参数（输入方式、换人时限）
3. 继续正常流程

## Anti-Patterns（禁止的模式）

### ❌ 错误模式 1：分步询问

```
用户：审查家政合同
AI：请问您想审查什么类型的合同？
用户：月嫂
AI：请问输入方式？
用户：文本粘贴
AI：请问试用期天数？
```

**问题：** 分步询问会打断工作流，应该一次性收集所有参数。

### ❌ 错误模式 2：中途确认

```
AI：分析完成，发现5个风险条款，是否继续生成报告？
用户：继续
AI：报告已生成，是否保存到文件？
```

**问题：** 不要中途请求确认，应该自动完成所有步骤。

### ❌ 错误模式 3：错误时停止

```
AI：解析失败，无法继续审阅。
```

**问题：** 应该记录错误并继续处理可识别部分，不要完全停止。

### ❌ 错误模式 4：未使用默认值

```
AI：您未指定试用期天数，请输入。如果不指定，我无法继续。
```

**问题：** 应该使用默认值并提示用户，不要停止等待输入。

## 常见条款修订对照示例

### 1. 试用期与退费

**原条款（中介版）：**
> 乙方上户工作满7天后，若因甲方原因辞退，中介费不予退还。若乙方原因离职，扣除20%服务费后退还余款。

**建议修订：**
> **条款 3.1 [试用期]**：本合同前 3个自然日 为无理由试用期。试用期内，甲方有权随时解除合同，且无需支付任何形式的违约金或赔偿金。
>
> **条款 3.2 [全额退费]**：在试用期内，若因乙方个人原因（包括但不限于身体健康、技能不符、个人意愿）或中介方推荐不实导致合同终止，丙方（中介）必须在3个工作日内 **无条件全额退还** 甲方缴纳的所有服务费用，不得扣除任何名目的手续费、匹配费。

### 2. 意外伤害责任

**原条款（中介版）：**
> 乙方在甲方家中工作期间发生意外伤害，由甲方承担赔偿责任。

**建议修订：**
> **条款 5.1 [责任自负]**：乙方系完全民事行为能力人，应具备与其工作内容相适应的安全防范意识。在服务期间，凡因乙方**自身健康原因**（如突发疾病、既往病史发作）或**违反操作规程**（如违规使用电器、攀高作业等）导致的人身损害，责任概由乙方自行承担。
>
> **条款 5.2 [有限补偿]**：甲方仅在存在**法定过错**且该过错直接导致乙方损害的情况下，依生效法律文书承担相应的过错责任。甲方出于人道主义垫付的任何费用，均视为对乙方的借款，有权在后续结算中全额追偿。

### 3. 隐私与监控

**原条款（中介版）：**
> 乙方不得泄露甲方隐私。

**建议修订：**
> **条款 6.1 [隐私禁区]**：乙方承诺在服务期间及离职后，严禁以任何形式（口头、书面、网络发布）泄露甲方的家庭住址、成员身份、生活作息、联系方式及家庭内部谈话内容。
>
> **条款 6.2 [禁止拍摄]**：未经甲方书面许可，乙方严禁在甲方住所内进行拍照、录像或录音。严禁将涉及甲方家庭成员（尤其是未成年人）的照片/视频上传至微信朋友圈、抖音、快手等任何社交媒体平台。
>
> **条款 6.3 [违约责任]**：若违反上述隐私条款，乙方需支付违约金人民币 **50,000元**，并赔偿因此给甲方造成的一切直接及间接损失。

### 4. 休假与加班

**原条款（中介版）：**
> 法定节假日加班，甲方需支付双倍工资。

**建议修订：**
> **条款 4.1 [节假日定义]**：本合同所指"法定节假日"不包括调休日期及周末连休，指元旦、春节、清明节、端午节、劳动节、中秋节和国庆节总计13日。
>
> **条款 4.2 [加班费计算]**：法定节假日加班费基数按【月基本工资 ÷ 26】计算。

### 5. 突发事件处理

**原条款（中介版）：**
> 月嫂服务员在您家工作时，若出现身体不适或其他疾病，发生意外事件，请您务必发扬人道主义精神，及时灵活做出处理，并通报月嫂家政服务机构。

**建议修订：**
> **条款 7.1 [突发事件]**：乙方若在工作中出现身体不适或其他疾病，或发生意外事件，甲方应及时通知中介公司，由中介公司负责处理，甲方不承担任何医疗及赔偿责任。

### 6. 定金退还

**原条款（中介版）：**
> 如果甲方在试用期内解除本合同，丙方（中介）有权扣取甲方定金1000元作为乙方误工补偿费，剩下全部退还甲方。

**建议修订：**
> **条款 9.1 [定金退还]**：如果甲方在试用期内解除本合同，丙方（中介）应全额退还甲方已支付的定金及预付款。

## 谈判话术库

### 高优先级话术

| 争议点 | 谈判话术 |
|--------|----------|
| 无过错不赔偿 | "正因为阿姨不容易，才要用合同把责任划清楚，免得以后出了事大家都说不清楚。无过错的情况下，雇主不应该承担赔偿责任。" |
| 隐私违约金 | "家里有老人小孩，隐私泄露的后果很严重。5万元违约金不是为了罚钱，是为了让对方知道这事的严重性。" |
| 试用期退费 | "试用期就是用来试的，不适合当然要退全款。扣钱的意思就是中介不自信，不敢保证服务质量。" |
